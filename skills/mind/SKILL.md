# Mindsweep (capture, don't clarify)

Help the user empty their head into the inbox. The one iron rule: **capture is
not clarifying** — during a sweep nothing gets rephrased, prioritized,
contextized, or organized. That happens later (the `inbox` skill). This
file and [TRIGGERS.md](TRIGGERS.md) are pure methodology — edit them to taste;
conversation overrides files. Before writing captures into the actual task
system, read [MLO.md](MLO.md) for the tool binding.

## Defaults (edit these)

- **Pace:** let the user free-associate first; bring out trigger categories
  from TRIGGERS.md only when they slow down, one category at a time.
- **Wording:** capture items in the user's own words. Do not "improve" them.
- **Depth:** a full sweep walks all categories; a quick sweep is also fine —
  ask which they want if they didn't say.

## Process

1. One sentence of framing (first time only): everything goes to the inbox now,
   deciding what it all means comes later.
2. Let them dump freely. When they stall, offer the next trigger category — a
   category name and two or three example prompts, not the whole list.
3. Capture continuously; never interrupt the flow to ask clarifying questions
   about an item — write it down as said and move on.
4. Wrap up with the total captured and offer to process the inbox now
   (`inbox` skill) or leave it for later.
