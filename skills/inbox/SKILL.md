# Inbox processing (GTD clarify & organize)

Walk every item in the inbox through the clarify flow and file it. This file is
pure methodology — a **base process, not dogma**; edit the Defaults to match
your flavor of GTD, and anything said in conversation overrides it. Before
touching the actual task system, read [MLO.md](MLO.md) for how each operation
binds to the tools.

## Defaults (edit these)

- **Interaction mode:** `confirm-batch` — clarify every item silently, present
  one compact plan (item → fate, one line each), apply after a single
  confirmation. <!-- alternative: `one-by-one` — ask about each item as you go -->
- **Two-minute rule:** items doable in under ~2 minutes are collected into a
  "do these now" list shown at the end, not filed.
- **Someday/Maybe:** kept on a dedicated list; propose creating it if missing
  (never create silently).
- **Reference:** non-actionable keep-worthy info is attached to the item it
  supports, or kept on a reference list.
- **Waiting-for:** delegated items are marked as waiting-for; if the system has
  no waiting-for convention yet, flag them in the plan and ask.

## Process

1. List the inbox. Empty → say so, stop.
2. Clarify each item with the standard questions — *what is it? is it
   actionable? what's the desired outcome? what's the very next physical
   action?*:
   - **Not actionable** → trash, Someday/Maybe, or reference (per Defaults).
   - **Actionable, under 2 minutes** → "do these now" list.
   - **Actionable, single step** → rewrite as a concrete verb-first next
     action, assign a context, file it where it belongs (or leave it at the
     top level if nothing fits).
   - **Multi-step outcome** → make it a project: outcome-phrased name, first
     next action(s) underneath, filed under the appropriate area.
   - **Someone else's move** → waiting-for treatment per Defaults.
3. Present the plan; apply once confirmed.
4. Close with counts (filed / projects created / deferred / deleted), the
   "do now" list, and anything left undecided.
